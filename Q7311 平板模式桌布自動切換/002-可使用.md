## Q

可使用.jxl作為桌布嗎？

## A

直接回答您的問題：**目前不建議，且大多數情況下無法直接使用。**

在 Windows 10 或 11 上，原生的桌布設定以及我剛剛提供給您的 PowerShell 腳本（使用 `SystemParametersInfo` API），都**不支援**直接讀取 `.jxl` (JPEG XL) 格式作為桌布。

如果您嘗試將 `.jxl` 檔案路徑放入腳本中，結果通常會是**黑畫面**或桌布完全沒有變化。

### 為什麼不能用？

1.  **API 限制**：Windows 用來更換桌布的底層指令（API）非常老舊，它最穩定支援的是 `.bmp`、`.jpg` 和 `.png`。即使您安裝了 JXL 的解碼器（Codec），這個底層指令通常還是看不懂 JXL 檔案。
    
2.  **系統支援度**：雖然 Windows 11 24H2 開始對 JXL 有較好的支援（如相片 App），但「桌面背景」這塊功能尚未完全跟上。
    

### 建議解決方案

為了讓您的 Q7311 自動旋轉換圖功能穩定運作，請採取以下最簡單的作法：

**將 `.jxl` 轉檔為 `.png` 或 `.jpg`**

*   **如果在此意畫質（無損）**：請轉存為 **.png**。PNG 支援無損壓縮，畫質與 JXL 無異，且所有 Windows 版本都能完美讀取。
    
*   **如果在此意檔案大小**：請轉存為高品質的 **.jpg**。
    

### 該如何修改腳本？

只要您將圖片轉檔後，記得回去修改 `AutoWallpaper.ps1` 裡面的副檔名即可：

```
# 修改為 .png 或 .jpg
$LandscapePath = "C:\Wallpapers\Landscape.png" 
$PortraitPath = "C:\Wallpapers\Portrait.png"

```

**小結：** 為了避免腳本執行錯誤或黑畫面，請務必先將 JXL 轉檔。
